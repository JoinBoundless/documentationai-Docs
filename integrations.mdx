## Setting Up Webhooks

Webhooks enable real-time notifications when documents change. Configure them to trigger actions in external services.

<Steps>
  <Step title="Create Webhook" icon="plus">
    Navigate to your project settings and select `Integrations` > `Webhooks`.
  </Step>
  <Step title="Add Endpoint" icon="link">
    Enter your webhook URL, such as `https://your-webhook-url.com/joinboundless`.
  </Step>
  <Step title="Select Events" icon="settings">
    Choose events like `doc.updated` or `doc.published`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and save the configuration.
  </Step>
</Steps>

Handle incoming webhooks in your service:

<CodeGroup tabs="Node.js,Python">
  ```javascript
  const express = require('express');
  const app = express();
  app.use(express.json());

  app.post('/joinboundless', (req, res) => {
    const { event, doc_id, changes } = req.body;
    console.log(`Event: ${event}, Doc: ${doc_id}`);
    // Notify team via Slack or update GitHub
    res.status(200).json({ status: 'received' });
  });

  app.listen(3000);
  ```
  ```python
  from flask import Flask, request, jsonify

  app = Flask(__name__)

  @app.route('/joinboundless', methods=['POST'])
  def webhook():
      data = request.json
      event = data.get('event')
      doc_id = data.get('doc_id')
      print(f"Event: {event}, Doc: {doc_id}")
      # Integrate with Slack or GitHub
      return jsonify({'status': 'received'}), 200

  if __name__ == '__main__':
      app.run(port=3000)
  ```
</CodeGroup>

## Custom API Connections

Build advanced integrations using JoinBoundless API endpoints.

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://api.example.com/v1/docs/{doc_id}/integrate \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "service": "custom",
      "config": {"endpoint": "https://your-service.com"}
    }'
  ```
  ```javascript
  const response = await fetch('https://api.example.com/v1/docs/{doc_id}/integrate', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      service: 'custom',
      config: { endpoint: 'https://your-service.com' }
    })
  });
  ```
</Request>

<Response tabs="200,400">
  ```json
  {
    "success": true,
    "integration_id": "int_123abc",
    "status": "active"
  }
  ```

  ```json
  {
    "error": "Invalid API key",
    "code": "auth_failed"
  }
  ```
</Response>

<ParamField path="doc_id" param-type="string" required="true">
  The unique document identifier.
</ParamField>

<ParamField body="service" param-type="string" required="true">
  Target service name, e.g., `custom`.
</ParamField>
